# 🎵 Melodic - 공연 통합 지원 서비스 기획서

## 1. 프로젝트 개요
**Melodic**은 소규모 공연 및 콘서트를 위한 **모바일 티켓팅 및 관객 소통 플랫폼**입니다.
복잡한 가입 절차 없이 간편하게 예매를 진행하고, 모바일 티켓을 발급받으며, 공연 전 설렘을 공유할 수 있는 응원 게시판 기능을 제공합니다.

---

## 2. 사용자 페르소나 (Target Audience)

### 👤 관객 (Audience)
- **니즈**: 간편한 예매, 내 티켓의 안전한 보관, 공연 정보 확인, 공연진에게 응원 메시지 전달.
- **특징**: 복잡한 회원가입을 꺼리며, 모바일 환경에서 모든 것을 해결하고 싶어 함.

### 🎭 공연진/관리자 (Performer / Admin)
- **니즈**: 실시간 입금 확인 및 티켓 발급, 관객 명단 관리, 공연 정보(타임라인, 셋리스트) 전달.
- **특징**: 공연 준비로 바빠 자동화된 입금 확인 및 티켓 관리 시스템이 필요함.

---

## 3. 핵심 기능 (Key Features)

### 🎫 1. 간편 예매 및 모바일 티켓 (Smart Ticketing)
*종이 티켓 없이, 링크 하나로 해결하는 입장 시스템*
- **예매 신청**: 별도 가입 없이 이름과 연락처만으로 예매 신청 (`/reserve`).
- **입금 확인 시스템**: 관리자가 입금 내역을 확인(또는 시뮬레이터 연동)하면 자동으로 예매 확정.
- **매직 링크 티켓**: 예매가 확정되면 고유 토큰이 포함된 링크(`/?auth=TOKEN`)가 생성됨. 이 링크를 통해 접속하면 즉시 **QR 코드 티켓**이 활성화됨.
- **입장 관리**: 공연 당일, QR 코드를 제시하여 입장 확인.

### 📢 2. 디지털 리플렛 (Digital Leaflet)
*종이 팜플렛 대신, 스마트폰 속 가이드*
- **공연 타임라인**: 입장 시간, 1부/2부 시작 시간, 인터미션 등을 타임라인 형태로 시각화하여 제공.
- **셋리스트 정보**: 1부와 2부의 곡명 및 아티스트 정보를 미리 확인 가능 (`/info`).
- **장소 및 일시**: 지도 아이콘과 함께 직관적인 공연 정보 제공.

### 💬 3. 응원 게시판 (Cheering Board)
*공연의 설렘을 공유하는 공간*
- **자유로운 소통**: 로그인 없이 닉네임만으로(또는 티켓 소지자로서) 응원 메시지 작성 가능 (`/board`).
- **포스트잇 감성**: 메모지 형태의 UI로 아날로그 감성 구현.
- **공개/비공개 설정**: 작성자가 원할 경우 특정 대상에게만 보이도록 설정 가능 (기능 로직 내재).

### 🛠 4. 관리자 대시보드 (Admin Dashboard)
*예매부터 발권까지 원스톱 관리*
- **입금 시뮬레이터**: 테스트 환경에서 가상의 입금 알림을 발생시켜 시스템 동작 확인.
- **자동 매칭 시스템**: 입금자명과 금액을 분석하여 대기 중인 예약자와 자동 매칭 및 승인.
- **티켓 링크 발급**: 승인된 예약자에 대한 고유 입장 링크 생성 및 복사 기능.

## 4. 사용자 흐름 (User Flow)

### Scenario A: 관객의 예매 여정
1. **탐색**: 공연 홍보물을 통해 예매 페이지 링크로 접속 (`/reserve`).
2. **신청**: 이름, 연락처 입력 후 '예매 신청하기' 클릭.
3. **입금**: 안내된 계좌번호로 티켓 금액(20,000원) 입금.
4. **수령**: (관리자 승인 후) 문자로 도착한 **티켓 링크** 클릭.
5. **확인**: 메인 화면(`Home`)에 접속되며, 본인 이름과 QR코드가 박힌 **디지털 티켓** 확인. "입장 시 보여주세요" 문구 확인.

### Scenario B: 공연 당일 관객 경험
1. **입장**: 공연장 도착 후 메인 화면의 QR 티켓 제시.
2. **정보 확인**: `/info` 탭에서 오늘의 셋리스트 확인. "아, 1부 엔딩이 이 곡이구나!"
3. **소통**: 대기 시간 동안 `/board` 탭에서 "오늘 공연 너무 기대돼요!" 응원글 작성.
4. **관람**: 타임라인에 맞춰 입장 및 관람.

### Scenario C: 관리자의 티켓 처리
1. **모니터링**: `/admin` 페이지에서 실시간 예약 신청 현황 확인.
2. **입금 확인**: 실제 통장 입금 내역과 대조 (또는 시뮬레이터로 테스트).
3. **승인**: 입금 확인 시 '승인' 버튼 클릭 또는 자동 매칭.
4. **발송**: 생성된 '티켓 링크'를 복사하여 관객에게 문자/카톡 발송.

## 5. 메뉴 구조 (Information Architecture)

- **Home (`/`)**:
  - (비로그인 시) 닉네임 설정/입장 화면
  - (티켓 소지 시) 디지털 티켓(QR) + 공연 타임라인
  - (일반 입장 시) 공연 포스터 + 타임라인
- **Reservation (`/reserve`)**: 예매 신청 폼 (네비게이션 바 숨김 처리)
- **Concert Info (`/info`)**: 일시, 장소, 셋리스트(Part 1, 2)
- **Board (`/board`)**: 응원 메시지 목록, 글쓰기 모달
- **Admin (`/admin`)**: 입금 시뮬레이터, 예약 대장, 승인 관리
- **Performer Login (`/performer/login`)**: 관계자 로그인 (테스트 계정 지원)

## 6. 개발 및 기술적 특징
- **Frontend**: React (Vite), Framer Motion (애니메이션)
- **Backend/DB**: Firebase Firestore (실시간 데이터 동기화)
- **Authentication**: Firebase Auth + Custom Token System (URL 쿼리 파라미터 인증)
- **Styling**: CSS Modules (컴포넌트 단위 스타일링)
